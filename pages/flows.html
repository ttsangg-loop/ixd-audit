Flows & Patterns

Flows and patterns follow actions and paths that users can take based on this Miro Board. As per Atomic Design, this would be analogous to Templates although this analysis does not necessarily study how elements are brought together to craft page layouts.

Searching and Filtering

Project Workspace - Searching

This is when you search within the Project Workspace (no results found).

This is when you search within the Project Workspace (results found and highlighted).

Project Workspace - Filtering

This is when you filter for things e.g. participants within the Project Workspace..

When you filter for a status, the results are homogenous in colour due to our colour-coding of progress.

We use a "More Filters" dropdown to store other filters, including a nested Compliance Question filter.

Similar to the Participant filter, we provide a toggle-able filter view for your tasks, where you can switch between tasks of different statuses assigned to you.

Adding New Things

When you create a new element in the admin-side of things (e.g. not creating a new project but creating a new variable, a new user, etc...) you start by clicking "+ Add New ___" at the bottom. After filling out your fields, you are able to save (or cancel). A banner will appear as a confirmation of your successful creation of this item.

Deleting Things

When you choose to delete something (which will often be hidden in an overflow menu), you are prompted to confirm via modal. The button that indicates confirmation of this non-reversible state is red. Confirmation will often appear in the form of a banner although this behaviour is not consistent across the platform.

Editing/Adding a Library Answer

When you edit a library answer, you are taken into the "backroom" of the entry - you are able to tweak its tags, its location, add alerts, link other entries, etc... The alert feature is currently available only to library entries as well and is used as a way to flag usage of this entry across other projects.

Starting in the PWS workspace, you can see alerts that were previously written for the library entry in question - you can only edit the alert if you return to the library entry and edit via "Edit Entry".

Importing a File

When you choose to import a file (upon project creation or adding a file after), the system takes a bit of time to read the file. Then you are guided to a highlightable source view where you can auto-select all the system-read questions.

Elements that were not initially highlighted or flagged as prospective questions/sections/subsections can be manually highlighted and tagged. This tags all other elements that the system reads follows the same hierarchy. From here, you are able to change the type of question or tag, although it similarly applies to all subsequent sections unless you choose to toggle off the auto-select feature.

You are also able to set Compliance and Additional Answer sets here.

Note that there are some usability issues with hovering and clicking around to denote what the highlighted text is.

After highlighting and selecting the appropriate text, you can click "Import", which will pull up a confirmation modal of elements before the questions are brought to the PWS.

Additional answers/compliance answers

You are able to adjust the settings/mappings of Compliance Sets and Additional Answers in the project itself (post-import). Compliance Sets appear as dropdowns with values depending on the mapping. If you have not mapped an Additional Answer to the question, you will receive an error message if you try to select it.

If you have mapped an Additional Answer to your entry, you are able to type in it just as you would type in a main answer. If you "Clear Entry", you will remove everything, including the Additional Answer (and any Compliance Sets as needed). You are also able to remove the mapping through the entry as well.

Error States

Loopio uses error states and messaging through modals, preventing actions, etc... to explain why certain actions cannot be done and/or to stop users from brute forcing something if they have not input all required information.

If an Additional Answer has not been mapped to this entry, then users will receive an error message when they try to edit it.

When creating a new project, users are unable to continue creating the project (and closing the modal) without filling in the missing information. Error messaging explains what is required/why their request was denied.

When creating a Merge Variable, users are also given error messages to explain why their action could not be completed. Note that when the user begins to edit any field, all error messaging disappears (perhaps not the best due to lack of transparency on what actually changed).
